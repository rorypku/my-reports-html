Snowflake Situation Report

1. Executive View

结论：推进下一阶段研究，但不是追高结论。Snowflake 刚在 2026-05-27 披露 Q1 FY2027，产品收入同比加速到 34%，FY2027 产品收入指引从 27% 上修到 31%，股票在 2026-05-28 收盘跳涨约 36%。这不是普通 beat，而是市场锚点从“消费增长放缓、AI 被 Databricks/云厂商挤压”切换到“AI 正在重新加速核心数据平台消费”的一次急转弯。

本 SitRep 的 wrong-anchor 假设是：市场仍主要用短期产品收入增速和 EV/Sales 给 SNOW 定价，但真正值得验证的是 AI 数据工作流采用度是否会转化为高质量净消费增长与单位工作负载贡献毛利。若 Cortex Code、Snowflake Intelligence、Iceberg/Openflow/Native Apps 让 Snowflake 成为企业 agentic control plane，估值锚应逐步从“高波动消费数仓”迁移到“组织级数据/AI 工作流平台”。若 AI 只是低毛利推理入口，或开放格式削弱锁定，则这次重估会变成叙事陷阱。

5. SitRep Judgment

Classification: Advance to next-stage research.

按照 advancement standard，SNOW 满足进入下一阶段的三条门槛。第一，Wrong anchor：市场仍在用产品收入增速和 EV/Sales 处理公司，但管理层正在把经济机制推向 agentic control plane、AI 数据工作流与治理依赖。第二，Material upside：如果公司从高波动 consumption warehouse 变成企业 AI 工作流控制面，合理锚点应加入 contribution margin、workflow lock-in 和 long-duration net consumption，而不是只盯 next-quarter beat cadence。第三，Clear validation path：未来 2-3 个季度可用 Q2/Q3 product revenue guide beat、NRR/RPO、大客户增长、AI account expansion、product gross margin 和 FCF margin 直接验证或证伪。

6. What Would Confirm Or Break The Thesis

Confirming datapoints

  • Q2 FY2027 产品收入高于 $1.415-$1.420B 指引，并且 FY2027 指引继续上修，而不是只靠 Q1 一次性 Cortex Code 导入。
  • NRR 维持或超过 126%，RPO 增速继续高于产品收入增速，$1M+ 与 $10M+ 客户数继续扩大。
  • 管理层继续维持约 75% non-GAAP product gross margin 与 23% adjusted FCF margin，同时披露 AI 工作负载对 core consumption 有正向拉动。
  • Summit / product disclosures 显示 Snowflake Intelligence、Cortex Code、Natoma / MCP、SAP / OpenAI / AWS 合作形成可复制的 enterprise workflow。

Breaking datapoints

  • Q2/Q3 guidance cadence 回到普通 3% beat，且 FY2027 指引不再上修，说明 Q1 只是情绪与模型重置。
  • AI 产品使用增长但 gross margin 或 FCF margin 下修，证明 token path 价值捕获不足。
  • Databricks / hyperscaler / open table ecosystem 把 Snowflake 推回 structured-warehouse 后端，导致新 workload attach rate 不足。
  • 客户优化或迁移项目造成 NRR 下滑，$1M+ 客户增长放缓，RPO 与 product revenue 出现背离。

7. Recommended Next Diligence

  1. 建立 quarterly bridge：产品收入、RPO、NRR、$1M+ customers、$10M+ customers、AI accounts、gross margin、FCF margin，拆出“AI adoption → core consumption”的证据链。
  2. 做竞争访谈清单：数据工程团队、GSI、Snowflake/Databricks 双栈客户，验证 Iceberg 后锁定层到底落在治理/语义/协作还是落在可替换 compute。
  3. 更新 post-Q1 sell-side reset：等 Goldman / Morgan Stanley / Bernstein / BofA / UBS 业绩后报告，比较目标价变化和估值锚是否从 EV/Sales 切向 EV/FCF 或 Rule-of-X。
  4. 补齐买方原文：寻找 2026H1 基金信、conference notes、operator memos 中明确讨论 SNOW 持仓和 AI data cloud valuation 的材料。
  5. 做 valuation sensitivity：FY2028 产品收入增长 22%/26%/30%、FCF margin 22%/25%/28%、EV/Sales 8x/12x/16x 三维场景，判断 post-spike 后仍是否有足够 IRR。
